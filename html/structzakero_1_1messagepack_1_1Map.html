Detailed Description

Using the Map structure allows any MessagePack Object to be used as a "key" as well as a "value" in a map. The underling object_key and object_value can be accessed directly. However for the Map, it is recommended to only use the helper methods.

Objects can be tested to find out if they are Map by using Object::isMap() and converted into a Map with Object::asMap(). A Map can not be converted into an Object. However, an Object can be constructed using a Map and the utility methods can operate on Maps as if they were Objects.

◆ keyIndex()

size_t zakero::messagepack::Map::keyIndex ( const Object &  key) const
noexcept

Use this method to get the index of a key. The benefits of using an index is to avoid constantly searching the Map. Also, using the index is the only way of changing the key.

Note
If any key/value pair is inserted or removed, all indexes after changed location are no longer valid.
Example
zakero::messagepack::Map map;
map.set(Object{}, Object{});
size_t index = map.keyIndex(Object{});
map.keyAt(index) = Object{true);
map.valueAt(index) = Object{std::string("true"));
